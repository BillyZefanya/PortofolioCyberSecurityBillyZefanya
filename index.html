Red Teaming Offensive Security

Attack Lifecycle

01

Surface Analysis & Recon

Pemetaan fungsionalitas aplikasi pada endpoint /cart.php.

Analisis Teknis: Mengidentifikasi parameter code sebagai vektor input dinamis. Kurangnya validasi input memungkinkan injeksi karakter berbahaya ke query database di sisi server.

02

Fuzzing & Error Confirmation

Injeksi karakter khusus untuk memicu respon anomali.

Analisis Teknis: Payload tanda kutip tunggal (') menyebabkan kegagalan sintaks SQL. Aplikasi membocorkan pesan error teknis yang mengonfirmasi kerentanan terhadap Error-Based SQL Injection.

03

Database Enumeration

Eksploitasi terautomasi menggunakan SQLMap untuk memetakan DBMS.

Analisis Teknis: Eksekusi automasi berhasil memetakan DBMS target (MySQL >= 5.6) dan mengekstraksi nama database sistem dibimbing-shop secara akurat.

04

Sensitive Data Exfiltration

Ekstraksi kredensial dari tabel admins dan customers.

Analisis Teknis: Ditemukan kebocoran data sensitif (*Sensitive Data Exposure*). Kredensial pengguna dan admin yang tersimpan dalam format plain-text, melanggar standar keamanan penyimpanan data.

05

Impact Validation (Account Takeover)

Verifikasi dampak serangan melalui login administratif.

Analisis Teknis: Validasi dampak nyata dengan melakukan pengambilalihan akun (Account Takeover) menggunakan kredensial hasil eksfiltrasi. Akses penuh ke dashboard administratif berhasil didapatkan.

Mitigation Strategy

Parameterized Queries

Mencegah interpretasi input user sebagai perintah SQL melalui implementasi Prepared Statements.

Strong Password Hashing

Mengamankan data kredensial menggunakan algoritma hashing standar industri seperti Bcrypt atau Argon2.
